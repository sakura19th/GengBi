# 小说档案
标题：{{title}}
作者：{{author}}
主角：{{protagonist}}
简介：{{synopsis}}
世界观设定：{{world_setting}}
写作风格：{{writing_style}}

# 前文章节内容
{{chapters_text}}

# 提取要求
请从以下 9 个维度深入提取关键信息，每个维度尽可能多条（若前文中确实存在）。参考创意写作技巧，关注人物弧光、感官细节、张力升级、伏笔回收等专业要素。

**重要边界**：
- **剔除世界观底层**：存在拓扑/因果架构/时空本体论/信息认识论/价值论基础/生成动力学/叙事本体论等底层世界观元规则不在此提取（由独立流程提取），本流程只提取具体实体层面的信息
- **剔除主角心理档案**：主角的认知风格/人格结构/动机系统/防御机制/行为指纹/关系坐标/弧光/OOC红线由独立流程提取（见 protagonist_behavior 维度只记录主角的既有行为）

## 1. characters（人物）
- 姓名、身份、年龄、外貌特征（含标志性服饰/物品）
- 性格特征与性格弧光（从何种状态向何种状态转变）
- 核心动机（外在目标 vs 内在需求）
- 情感节拍（当前情绪状态、未释放的情感张力）
- 能力/技能/弱点（含超能力/战斗体系设定）
- 当前处境与未解决的内心冲突
- **剔除主角心理档案**：主角的认知风格/人格结构/动机系统/防御机制/行为指纹/关系坐标/弧光/OOC红线由独立流程提取，本维度只记录主角的既有行为（已做出的关键决策、已展现的能力、已发生的行为模式）

## 2. relationships（关系动态）
- 人物间关系类型（敌对/同盟/暧昧/师徒/竞争等）
- 关系演变轨迹（从何种关系发展为何种关系）
- 关键对话片段（体现关系本质的代表性对话）
- 权力结构（谁主导、谁服从、暗藏的博弈）
- 未挑明的情感暗线

## 3. locations（地点）
- 名称、描述、功能、地理位置关系
- 感官细节（视觉/听觉/嗅觉/触觉/味觉特征）
- 氛围营造（压抑/温馨/神秘/危险等基调）
- 功能象征意义（地点对情节/人物的隐喻作用）
- 未探索区域（伏笔性地点提示）

## 4. events（事件）
- 已发生的关键事件（按时间线排列）
- 因果链（事件 A 导致事件 B 的逻辑链）
- 张力升级阶梯（冲突如何逐步加剧）
- 转折点与反转（出人意料的情节转折）
- 时间线节点（关键时间点与时间跨度）

## 5. atmosphere（氛围感官）
- 整体基调（黑暗/轻松/悬疑/热血等）
- 感官描写模式（作者偏好的感官描写手法）
- 节奏特征（快节奏动作 vs 慢节奏抒情）
- 环境烘托手法（天气/光影/色彩如何暗示情绪）
- 情绪曲线（章节间的情绪起伏走向）

## 6. style（风格）
- 叙事视角（第一人称/第三人称限知/全知/多视角切换）
- 语言特色（用词偏好、句式长短、口语化程度）
- 节奏控制（场景切换频率、详略安排）
- 常用修辞手法（比喻/拟人/排比/反讽等）
- 对话风格（角色对话的语气、口头禅、信息密度）
- 描写偏好（重动作/重心理/重环境/重对话）

## 7. foreshadowing（伏笔回收）
- 已埋伏笔（尚未回收的暗示/线索/预言）
- 已回收伏笔（前文埋下、后文揭开的呼应）
- 悬念设置（未解之谜、待揭晓的秘密）
- 伏笔密度与回收节奏（作者埋伏笔的习惯模式）
- 可能的伏笔（容易被忽略但可能重要的细节）

## 8. plot_state（剧情状态）
- 当前剧情进展（最新章节结束时的局势）
- 主线推进度（主线目标完成百分比/阶段）
- 支线状态（各支线的进展与交汇点）
- 人物目标与冲突（当前各角色的短期/长期目标）
- 未解决的线索（悬而未决的问题/谜题）
- 主题意象（反复出现的象征物/主题词/核心命题）
- 下一章可能走向（基于前文逻辑的合理推测方向）

## 9. protagonist_behavior（主角既有行为）
- 已做出的关键决策（按时间线排列，含决策动机与后果）
- 已展现的能力与技能（含使用场景与限制）
- 已发生的行为模式（应对压力/冲突/亲密关系的典型行为）
- 已建立的人际关系动态（与各角色的互动模式）

# 输出格式
请输出严格的 JSON 数组，每个元素为一个 ContextEntry 对象，包含以下字段：
- uid：字符串，唯一标识（如 "char_1"、"loc_1"、"rel_1"、"atm_1"、"fore_1"）
- category：字符串，取值 characters/relationships/locations/events/atmosphere/style/foreshadowing/plot_state/protagonist_behavior
- key：字符串数组，关键词列表（用于显示和搜索）
- comment：字符串，简短说明（如"主角性格弧光"、"关键伏笔"）
- content：字符串，详细内容（最长 500 字，尽可能详尽）
- order：整数，排序权重（默认 100，数字越小越优先，关键信息设较低值）
- position：字符串，注入位置（before/after/at_depth）
- depth：整数，仅 at_depth 时有效（默认 4）
- role：字符串，消息角色（system/user/assistant，默认 system）

# 提取原则
- 信息密度优先：尽可能多地提取有价值信息，不要遗漏重要细节
- 创作导向：提取的信息应直接服务于续写质量（人物一致性、情节连贯、风格统一）
- 具体而非抽象：content 中引用具体文本细节（对话片段、场景描写），避免空泛概括
- 动态追踪：关注"变化"而非"静态"——人物如何变化、关系如何演变、局势如何发展
- 伏笔敏感：对前文中任何可能的暗示、象征、铺垫保持高度敏感
- 风格还原：提取足够的风格特征，确保续写能还原原文的语言质感

注意：
- 只输出 JSON 数组，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- content 字段可详尽展开，控制在 500 字以内
- 确保信息准确，不要编造前文中不存在的内容
- 目标提取数据量：尽可能丰富，总 token 数接近 5000
