# 验证任务
请从以下五个维度严格验证续写文本的质量。审查聚焦于用户输入指令的执行情况、续写文本与底层世界观及主角形象档案的一致性，不涉及大纲与卷级结构。

**再次强调：维度 4（主角一致性）与维度 5（世界观一致性）为必审项，每次审查都必须认真执行，不得因"看似无问题"而跳过。审查深度直接决定 passed 判定的可信度。**

## 1. consistency（一致性）
- 续写文本内部逻辑是否自洽
- 人物行为是否符合前文已建立的状态与动机
- 世界观设定是否与前文一致（若主角形象档案/世界观底层缺失，则仅审查文本内部自洽性）
- 叙事承诺是否被合理推进或兑现

## 2. style（文风）
- 叙事视角是否与前文统一
- 语言特色、句式、用词是否还原前文风格
- 修辞手法、描写偏好是否保持一致
- 对话风格是否与人物设定匹配

## 3. coherence（连贯性）
- 续写与前文衔接是否自然
- 叙事逻辑是否通顺无断裂
- 场景转换是否合理
- 时间线推进是否连贯

## 4. protagonist_consistency（主角一致性）【必审项】
**本维度为必审项，每次必须逐项完成以下 8 项自检并输出结论，不得跳过。** 参考主角形象档案进行 8 项一致性自检：
1. 性格一致性：主角是否有言行违背性格铁律或绝对不能做的事？
2. 动机回溯：主角的每个重大决定是否能追溯到动机优先级或核心恐惧？有无剧情奴隶化？
3. 对话剥离测试：去掉角色名后台词是否仍能辨认出是主角？
4. 防御机制检查：面对压力时主角是否使用了设定的防御机制？
5. 依恋模式检查：亲密关系场景中主角是否表现出设定的依恋类型特征？
6. 创伤触发检查：涉及类似创伤场景时主角的应激反应是否符合设定？
7. 语言指纹检查：主角的用词/句式/称呼是否与设定一致？有无知识越界？
8. 身体语言检查：主角的情绪是否通过设定的身体语言表达？
**一票否决**：出现 critical 级别 protagonist_consistency 问题时 passed=false。若主角任何行为违反 OOC 红线（绝对不能做的事），或决策模式不符合校验优先级（性格铁律>动机优先级>核心恐惧>默认防御>剧情需要），该维度直接判 critical，passed=false

## 5. worldview_consistency（世界观一致性）【必审项】
**本维度为必审项，每次必须逐项完成以下检查并输出结论，不得跳过。** 对照底层世界观元描述（7 大维度参数化描述）严格审查：
- 正文中的事件是否符合底层世界观元规则（因果架构/时空规则/守恒律）？
- 是否有违反存在拓扑（existential_topology）的设定——如出现不该存在的存在层级？
- 是否有违反因果架构（causal_architecture）的设定——如因果链断裂、后果先于原因？
- 是否有违反时空本体论（spatio_temporal_ontology）的设定——如时空规则前后矛盾？
- 是否有违反信息认识论（information_epistemology）的设定——如角色获取不该知道的信息？
- 是否有违反价值论基础（axiological_foundation）的设定——如价值体系崩塌无交代？
- 是否有违反生成动力学（becoming_dynamics）的设定——如世界演变规律前后不一？
- 是否有违反叙事本体论（narrative_ontology）的设定——如叙事层规则被破坏？
**严格给分**：违反底层世界观元规则时该维度问题至少判 major 级别

# 输出格式
请输出严格的 JSON 对象，包含以下字段：
- summary：字符串，评审总体结论。**必须包含以下两个固定标记段落，每段至少一句话明确陈述该维度的审计结论**：
  - `【主角一致性审计】`：明确陈述是否对照主角形象档案完成 8 项自检、是否发现 OOC 问题（如"已对照档案完成 8 项自检，主角言行符合性格铁律与动机优先级，未发现 OOC"或"第3段主角对话违背语言指纹设定，属于 OOC"）
  - `【世界观一致性审计】`：明确陈述是否对照世界观底层元描述完成 7 维度检查、是否发现违规（如"已对照元描述完成 7 维度检查，事件符合因果架构与时空规则，未发现违规"或"第2段角色获取了不该知道的信息，违反信息认识论"）
  - 若主角形象档案或世界观底层缺失，对应段落注明"档案缺失，无法审计"并说明原因
  - 其余部分概括续写文本的整体质量与其他维度的审查情况，以及对用户输入指令的执行情况
- issues：数组，发现的问题，每项含：
  - category：字符串，问题类别，取值 consistency/style/coherence/protagonist_consistency/worldview_consistency
  - severity：字符串，严重程度，取值 critical（致命，破坏一致性或逻辑）/ major（重要，明显影响质量）/ minor（轻微，可优化）
  - location：字符串，问题所在位置（段落编号或引用相关文本片段）
  - description：字符串，问题具体描述
  - suggestion：字符串，修改建议
- passed：布尔值，当不存在 critical 与 major 级别问题时为 true，否则为 false

# 验证原则
- 严格对照：所有判断必须基于续写文本、主角形象档案与世界观底层，不得凭主观偏好
- 客观公正：只指出真实存在的问题，不夸大也不遗漏
- 具体可改：description 应引用具体文本，suggestion 应可操作
- 严重分级：critical 仅用于破坏故事一致性或逻辑的致命问题
- **必审原则**：主角一致性（维度 4）与世界观一致性（维度 5）为必审项，无论续写文本长短、无论其他维度是否发现重大问题，每次都必须严格对照主角形象档案与世界观底层元描述完成审查，并在 summary 中输出专门的审计结论段落。不得以"无明显问题"为由省略审查过程与结论

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无问题时 issues 为空数组
- passed 字段必须严格依据 critical 与 major 问题数量判定
- **summary 必须包含 `【主角一致性审计】` 与 `【世界观一致性审计】` 两个固定标记段落，缺失任一标记段落视为审查不合格**
- 主角一致性与世界观一致性维度的 issues 条目（若有）必须使用对应的 category 值（protagonist_consistency/worldview_consistency），不得错误归类到 consistency
- 确保 JSON 语法严格合法，可直接被解析
