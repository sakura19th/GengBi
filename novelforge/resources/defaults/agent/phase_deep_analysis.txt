# 分析任务
请基于上述小说档案与前文章节文本，进行**详细、非模板化**的分析。所有分析必须基于前文实际内容，不得套用通用模板或泛化结论。需从以下维度逐项剖析：

**重要：文本使用说明**
- 所有维度的分析均基于上述章节文本
- 切分模式下，每块仅含部分章节，分析基于当前块文本，最终通过增量合并产出完整报告

1. 剧情排布规律：前文章节如何切分（按场景/按时间/按视角/按事件），每章内部结构（起承转合的实际呈现方式），情节推进节奏，章节篇幅分布与波动
2. 张力曲线规律：前文张力如何起伏，高潮分布位置，松紧交替模式，是否存在周期性张力节奏
3. 钩子设置规律：章节结尾钩子/悬念的设置手法、出现频率、钩子类型（悬念/反转/冲突/信息差/情感）
4. 文风分析：叙事视角（人称/限知/全知）、语言风格（古风/白话/网文/书面）、修辞偏好、句式特征（长短/对仗/排比）、词汇层级、描写密度
5. 对话分析：对话数量占比、对话风格（文言/口语/方言/混杂）、对白与叙述配比模式、对话的功能（推进/塑造/铺垫）
6. 伏笔清单：已埋设（planted）/已回收（triggered）/进行中（ongoing），每项含 item/status/setup_chapter/importance(high/medium/low)
7. 常用梗/套路：前文反复使用的叙事套路、桥段、类型惯例，每项含 trope/description/frequency
8. 设定库：世界观/人物/物品/体系/势力/地点，每项含 category/name/description
9. 人物弧光规律：主要人物的价值观与关系变化轨迹
10. 反复元素：反复出现的意象、标志性短语、口头禅、场景符号
11. 故事状态：structure_position/tone/core_conflict_status/stakes/active_characters/plot_threads/unresolved_promises/world_state
12. 用户剧情输出需求解析：基于用户输入的剧情输出需求，提取后续续写必备要素（required_elements，后文必须出现的人物/事件/设定/情节节点）与必须着重强调的部分（emphasized_elements，需重点描写/强化呈现的要素），整体理解（interpretation）用户意图，并标注与前文设定的冲突（conflicts，若有）

# 输出格式
请输出严格的 JSON 对象（DeepAnalysis），包含以下字段：
- structure_position：字符串，当前故事在三幕/多幕结构中所处位置
- tone：字符串，当前整体基调与情绪氛围
- core_conflict_status：字符串，核心冲突的当前状态与推进程度
- stakes：字符串，当前各方赌注/代价/风险
- active_characters：数组，当前活跃人物，每项含 name/status/motivation
- plot_threads：数组，当前情节线，每项含 name/status/priority(high/medium/low)
- unresolved_promises：数组，未兑现的叙事承诺，每项含 description/setup_chapter
- world_state：字符串，当前世界状态
- plot_arrangement_analysis：字符串，剧情排布规律的详细分析
- chapter_structure_pattern：字符串，每章内部结构与篇幅分布规律
- tension_curve_pattern：字符串，张力曲线与高潮分布规律
- hook_patterns：字符串，钩子设置手法与频率规律
- style_analysis：字符串，文风分析（视角/语言/修辞/句式/词汇/描写密度）
- dialogue_analysis：字符串，对话分析（占比/风格/配比/功能）
- pacing_analysis：字符串，节奏与推进速度分析
- character_arc_patterns：数组，主要人物弧光规律，每项含 name/arc_trajectory/value_shift
- foreshadowing_inventory：数组，伏笔清单，每项含 item/status(planted/triggered/ongoing)/setup_chapter/importance(high/medium/low)
- common_tropes：数组，常用梗/套路，每项含 trope/description/frequency
- settings_database：数组，设定库，每项含 category/name/description
- recurring_elements：数组，反复元素，每项含 element_type/image
- key_phrases：数组，标志性短语/口头禅，每项含 phrase/context
- user_directive_analysis：对象，用户剧情输出需求解析，含 required_elements（数组，后文输出必备要素）/emphasized_elements（数组，必须着重强调的部分）/interpretation（字符串，整体理解）/conflicts（数组，与前文设定的冲突）

# 分析原则
- 忠于原文：所有判断必须基于前文实际内容，不得编造
- 动态追踪：关注"变化"——人物如何变化、冲突如何升级、伏笔如何演进
- 结构敏感：准确判断当前在整体叙事结构中的位置
- 伏笔敏感：对任何暗示、象征、铺垫保持高度警觉
- 风格还原：提取足够风格特征，确保续写能还原原文语言质感
- 非模板化：所有分析必须基于前文实际内容，不得套用通用模板或泛化结论；若前文无对应现象，应据实说明而非凑数
- 用户指令优先：所有分析必须将用户剧情输出需求视为后续续写的硬约束，提取的 user_directive_analysis.required_elements 与 emphasized_elements 将作为后续大纲生成的强制依据；若用户指令与前文设定冲突，应在 conflicts 中明确标注而非自行忽略

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无内容时使用空字符串或空数组
- 各类清单条目数受 analysis_depth 与 max_analysis_entries 共同约束
- 确保 JSON 语法严格合法，可直接被解析
