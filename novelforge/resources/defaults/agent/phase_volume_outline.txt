请综合参考前文深度分析结果（DeepAnalysis）、前文章节文本与用户续写指令，规划 N 章的卷大纲。核心要求如下：

- 参考 DeepAnalysis 中的张力曲线规律与钩子设置规律，规划跨章张力曲线（逐步升级至卷高潮后回落），每章设钩子
- 参考 DeepAnalysis 中的伏笔清单（优先回收 planted 伏笔）与设定库，在新卷中复用原设定、回收原伏笔、埋设新伏笔，并延续常用梗

# 输出格式
请输出严格的 JSON 对象（VolumeOutline），包含以下字段：
- volume_title：字符串，本卷标题
- volume_goals：字符串，本卷总体叙事目标
- plot_arrangement_analysis：字符串，本卷剧情排布规划（如何对应 DeepAnalysis 中的排布规律）
- pacing_plan：字符串，本卷节奏与张力曲线规划
- foreshadowing_plan：字符串，本卷伏笔回收与埋设计划
- chapter_count：整数，章节数（必须等于输入的 chapter_count）
- chapters：数组，各章规划，数组长度必须严格等于 chapter_count，每项含：
  - index：整数，章节序号（从 1 开始）
  - title：字符串，章节标题
  - summary：字符串，本章剧情摘要
  - plot_role：字符串，本章叙事功能，必须为以下六值之一（单选，严禁组合/拼接）：起、承、转、合、高潮、过渡
  - key_events：数组，本章关键事件
  - characters_involved：数组，本章涉及人物
  - foreshadowing：字符串，本章伏笔埋设或回收说明
  - chapter_hook：字符串，本章结尾钩子
  - target_words：整数，本章目标字数（应接近 target_words_per_chapter）

# 规划原则
- 一致性优先：新卷规划必须与 DeepAnalysis 中的人物状态、设定、世界状态、文风保持一致
- 冲突驱动：每章应有明确的冲突与目标，避免无张力过渡
- 节奏把控：跨章张力曲线应逐步升级至卷高潮后回落，符合 DeepAnalysis 中的张力曲线规律
- 伏笔回收：优先回收 DeepAnalysis 中 status 为 planted 的伏笔，同时可埋设新伏笔
- 风格统一：章节切分方式、内部结构、文风、对话密度应与 DeepAnalysis 中的分析结果一致
- 用户意图：若用户指令非空，应在规划中体现用户意图

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无内容时使用空字符串或空数组
- chapters 数组长度必须严格等于 chapter_count
- 确保 JSON 语法严格合法，可直接被解析
