你是一位专业的小说分析师。请深入分析以下小说前文章节，产出结构化的 STORY STATUS SNAPSHOT（故事状态快照），用于支撑后续多轮 Agent 续写的一致性与连贯性。请以严格 JSON 格式输出。

# 小说档案
标题：{{title}}
作者：{{author}}
主角：{{protagonist}}
简介：{{synopsis}}
世界观设定：{{world_setting}}
写作风格：{{writing_style}}

# 前文章节内容
{{chapters_text}}

# 分析任务
请基于上述小说档案与前文章节文本，从结构、基调、冲突、人物、情节线、承诺、伏笔、世界状态、风格等维度综合分析当前故事状态，产出一份结构化的 STORY STATUS SNAPSHOT。

# 输出格式
请输出严格的 JSON 对象，包含以下字段：
- structure_position：字符串，当前故事在三幕/多幕结构中所处位置（如"第一幕尾声"、"中点转折"、"高潮前夕"等）
- tone：字符串，当前整体基调与情绪氛围（如"压抑悬疑"、"热血昂扬"、"温情治愈"等）
- core_conflict_status：字符串，核心冲突的当前状态与推进程度
- stakes：字符串，当前各方赌注/代价/风险（主角失败会失去什么）
- active_characters：数组，当前活跃人物，每项含：
  - name：字符串，人物姓名
  - status：字符串，当前状态（处境/情绪/立场）
  - motivation：字符串，当前动机（外在目标 vs 内在需求）
- plot_threads：数组，当前情节线，每项含：
  - name：字符串，情节线名称
  - status：字符串，进展状态（如"未启动/推进中/受阻/即将交汇/已收束"）
  - priority：字符串，优先级（high/medium/low）
- unresolved_promises：数组，未兑现的叙事承诺，每项含：
  - description：字符串，承诺内容描述
  - setup_chapter：字符串，埋设该承诺的章节标识
- foreshadowing_tracker：数组，伏笔追踪，每项含：
  - item：字符串，伏笔内容
  - status：字符串，状态，取值 planted（已埋设未触发）/ triggered（已触发回收）/ ongoing（持续推进中）
- world_state：字符串，当前世界状态（势力格局/环境变化/时间推进等）
- style_profile：字符串，前文风格画像（叙事视角、语言特色、节奏、修辞偏好等，用于续写风格还原）

# 分析原则
- 忠于原文：所有判断必须基于前文实际内容，不得编造
- 动态追踪：关注"变化"——人物如何变化、冲突如何升级、伏笔如何演进
- 结构敏感：准确判断当前在整体叙事结构中的位置
- 伏笔敏感：对任何暗示、象征、铺垫保持高度警觉
- 风格还原：提取足够风格特征，确保续写能还原原文语言质感

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无内容时使用空字符串或空数组
- 确保 JSON 语法严格合法，可直接被解析
