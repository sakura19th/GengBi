# ⚠️ 重写边界（最关键硬约束，必须严格遵守）
- **不推进剧情**：重写不得让故事时间线向前推进。重写后的章节事件范围必须与原章节一致（同一时间段、同一场景边界），不得发生原章节时间范围之后的新事件
- **不续写下一章**：重写不是续写。不得在原章节末尾追加后续情节、不得过渡到下一章场景、不得为后续章节做铺垫性推进（保留原章节结尾的开放性/封闭性不变，除非用户明确要求修改结尾性质）
- **不新增场景**：不得引入原章节未涉及的新场景/新地点/新时间点。用户要求的「新增内容」仅限于当前章节已有场景内的描写/对话/细节补充，不得开辟新场景
- **不引入新角色**：不得引入前文未出现且原章节未提及的新角色。可在已有角色范围内调整戏份/对话/描写
- **不解决未决伏笔**：不得在重写中提前解决当前章节未解决的伏笔（除非用户明确要求且该解决不推进剧情）
- **允许的调整**：允许修改对话措辞/调整描写细节/补充内心独白/修改结尾性质（如悲剧→开放式）/调整场景内节奏，但所有调整必须限定在原章节事件范围内

**判断标准**：重写后的章节读完，读者获得的故事信息量与原章节**基本相当**（可有细节增减，但不得新增剧情推进）。若重写后读者知道了原章节未涉及的后续发展，即视为违规推进剧情。

# 分析任务
请综合参考当前章节正文、用户重写需求、前文上下文、世界观底层与主角形象档案，输出「新章节生成详细需求」。**该需求将直接作为后续生成步骤的核心指令，须具体、可操作、可执行。**

分析原则：
- **严格遵循用户指令**：用户重写需求视为硬约束，不得在分析中擅自添加用户未要求的意外事件、新角色、新剧情或新场景。分析仅针对用户需求与当前章节已有内容，不得扩展超出用户指令范围的创作建议。
- **重写边界最高优先**：不推进剧情是最高硬约束，优先级高于用户需求。若用户需求涉及推进剧情（如「续写下一章」「让主角离开当前场景前往新地点」），在 conflicts 中标注并建议用户改用续写模式
- **用户需求优先**：用户重写需求视为硬约束（在不推进剧情前提下），优先级高于通用原则，但低于重写边界、主角 OOC 红线与世界观底层规则
- **基于实际内容**：分析须基于当前章节与前文的实际内容，不得套用通用模板或泛化结论
- **一致性硬约束**：主角 OOC 红线与世界观底层规则是硬约束，重写需求不得违反
- **保留优质元素**：当前章节的精彩对话、有效伏笔、出色描写等优质元素应保留，仅在薄弱处修订
- **针对性强**：每条需求应针对具体问题或用户指令，可操作可执行，且须限定在原章节事件范围内
- **剧情排布保全**：重写须在不影响原来剧情排布发展的前提下满足用户诉求。原章节的事件序列、场景顺序、关键转折点、伏笔布局、人物登场退场节奏均为须保全的剧情骨架，用户需求的落实仅限在不破坏该骨架的前提下进行细节调整、描写修订、对话改写、氛围强化等。若用户诉求与原剧情排布冲突，在 conflicts 中标注并建议用户改用续写模式
