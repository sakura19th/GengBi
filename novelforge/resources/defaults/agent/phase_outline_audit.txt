请对每个启用的审计维度进行评分与问题诊断，并基于审计结果修订原大纲，产出终稿 VolumeOutline。各维度定义如下：

- consistency（一致性）：与原前文设定/人物/世界状态是否一致
- pacing（节奏）：张力曲线/起承转合分布/快慢交替是否合理
- engagement（吸引力）：每章钩子是否吸引人、是否有拖沓
- structure（结构）：三幕结构比例是否合理、卷高潮位置
- coherence（连贯性）：章节衔接、伏笔回收链条
- foreshadowing（伏笔）：planted 伏笔回收情况、新伏笔埋设
- characters（人物）：人物弧光、动机一致性、关系发展
- style（文风）：文风/语言特色/叙事视角是否与前文统一，是否符合卷大纲设定的 writing_style
- protagonist_consistency（主角一致性）：大纲中主角的行为/决策/目标是否符合 protagonist_profile 中的性格铁律、动机优先级、核心恐惧、OOC红线？**一票否决**：该维度 score ≤ 4 时整体 passed=false。若大纲中主角任何行为违反 OOC 红线（绝对不能做的事），或决策模式不符合校验优先级（性格铁律>动机优先级>核心恐惧>默认防御>剧情需要），该维度直接 0 分，整体 passed=false
- worldview_consistency（世界观一致性）：大纲中的事件/设定是否符合 world_ontology 中的因果架构、时空规则、守恒律？**严格给分**：违反底层世界观元规则时该维度 score ≤ 3
- user_directive_compliance（用户指令遵从性）：修订后的大纲是否满足 user_directive_analysis.required_elements 中的所有必备要素？是否着重体现了 emphasized_elements 中的强调部分？**严格给分**：未满足 required_elements 任一项时该维度 score ≤ 3；conflicts 中的冲突未在修订大纲中明确处理的也应低分
- custom_rules_compliance（自定义设定遵从性）：修订后的大纲是否满足每条自定义规则的 requirement 与 audit_criteria？**严格给分**：违反 severity=critical 的规则时该维度 score ≤ 3，整体 passed=false；违反 severity=major 的规则时该维度 score ≤ 5
- rigid_ai_text（刻板AI文本禁令）：大纲中是否存在典型 AI 生成套路文本？**严格给分**：出现 3 处以上典型 AI 痕迹（句式套路/词汇黑名单/描写方法/对白设计/AI痕迹/认知越界）时该维度 score ≤ 5；出现 5 处以上或整段为 AI 套路文本（如整段排比/列举/哲理总结）时 score ≤ 3，整体 passed=false

仅审计上述启用的维度，未启用的维度可跳过。

# 输出格式
请输出严格的 JSON 对象（OutlineAuditReport），包含以下字段：
- dimensions：数组，各启用维度的审计结果，每项含：
  - dimension：字符串，维度名（consistency/pacing/engagement/structure/coherence/foreshadowing/characters/style/protagonist_consistency/worldview_consistency/user_directive_compliance/custom_rules_compliance/rigid_ai_text）
  - score：整数，该维度评分（1-10，10 为最佳）
  - issues：数组，该维度发现的具体问题列表（每项为字符串）
  - suggestions：数组，针对该问题的具体修订建议列表（每项为字符串）
- overall_assessment：字符串，整体评估说明
- passed：布尔，是否整体通过（各维度评分均达标且无严重问题则为 true）
- revised_outline：对象，修订后的完整 VolumeOutline（含全部 chapters），结构与卷大纲模板一致，包含 volume_title/volume_goals/plot_arrangement_analysis/pacing_plan/foreshadowing_plan/chapter_count/chapters 字段，chapters 每项含 index/title/summary/plot_role(必须单选：起/承/转/合/高潮/过渡，严禁组合)/key_events/characters_involved/foreshadowing/chapter_hook/target_words

# 审计原则
- 客观评分：评分应基于具体证据，避免主观偏好
- 问题具体：每个 issue 应明确指出问题所在章节与具体表现
- 建议可操作：每个 suggestion 应可直接落地到修订后的 outline
- 修订到位：revised_outline 必须切实改善审计中发现的问题，而非简单复制原大纲
- 用户指令优先：用户指令遵从性是硬约束，required_elements 未满足时该维度必须低分，不可因其他维度优秀而豁免

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无内容时使用空字符串或空数组
- revised_outline 必须是完整的 VolumeOutline（含全部 chapters，chapters 长度与原大纲一致）
- 确保 JSON 语法严格合法，可直接被解析
