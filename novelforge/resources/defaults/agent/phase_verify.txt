# 验证任务
请对照前文快照与续写大纲，从以下四个维度严格验证续写文本的质量：

## 1. consistency（一致性）
- 人物行为是否符合前文快照中的状态与动机
- 世界观设定是否与前文一致
- 情节线推进是否符合快照中的状态
- 伏笔回收是否符合 foreshadowing_tracker 的设定
- 叙事承诺是否被合理推进或兑现

## 2. style（风格）
- 叙事视角是否与 style_profile 一致
- 语言特色、句式、用词是否还原前文风格
- 修辞手法、描写偏好是否与前文统一
- 对话风格是否与人物设定匹配

## 3. structure（结构）
- 场景是否完整覆盖大纲规划
- 每个场景的目标-冲突-结局是否落实
- value_shift 与 exit_hook 是否有效呈现
- 整体节奏与张力曲线是否合理

## 4. engagement（吸引力）
- 场景是否具有足够张力与冲突
- 人物动机是否清晰可感
- 钩子是否有效吸引读者继续阅读
- 情感节拍是否能引发共鸣

# 输出格式
请输出严格的 JSON 对象，包含以下字段：
- summary：字符串，评审总体结论（概括续写文本的整体质量）
- issues：数组，发现的问题，每项含：
  - category：字符串，问题类别，取值 consistency/style/structure/engagement
  - severity：字符串，严重程度，取值 critical（致命，破坏一致性或逻辑）/ major（重要，明显影响质量）/ minor（轻微，可优化）
  - location：字符串，问题所在位置（场景编号、段落或引用相关文本片段）
  - description：字符串，问题具体描述
  - suggestion：字符串，修改建议
- passed：布尔值，当不存在 critical 与 major 级别问题时为 true，否则为 false

# 验证原则
- 严格对照：所有判断必须基于前文快照与大纲，不得凭主观偏好
- 客观公正：只指出真实存在的问题，不夸大也不遗漏
- 具体可改：description 应引用具体文本，suggestion 应可操作
- 严重分级：critical 仅用于破坏故事一致性或逻辑的致命问题

注意：
- 只输出 JSON 对象，不要包含任何解释性文字
- 不要使用 markdown 代码块标记
- 所有字段必须存在，无问题时 issues 为空数组
- passed 字段必须严格依据 critical 与 major 问题数量判定
- 确保 JSON 语法严格合法，可直接被解析
