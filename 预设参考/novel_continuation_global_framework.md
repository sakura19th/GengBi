## 一、底层世界观元描述（World Ontology Layer）

本层描述虚构世界在最底层如何运行，不涉及任何具体实体，只描述世界本身的运行律、存在结构和认知框架。它是所有上层设定（魔法体系、社会结构、历史事件）的**底层操作系统**。

### 1.1 存在拓扑（Existential Topology）

描述"存在"本身的结构与层级。

#### 1.1.1 存在层级（Being Hierarchy）

| 层级 | 类型 | 本体论地位 | 可变性 | 感知模式 |
|------|------|-----------|--------|----------|
| **物理层** | 物质/能量/时空 | 基础实在/派生/模拟/梦境 | 可变/恒定/周期性重置 | 直接感官/仪器测量/间接推断 |
| **生命层** | 生物/灵魂/数据/意识 | 物理层涌现/独立实体/寄生/投影 | — | 死亡后去向：消散/转层/轮回/存储/升格 |
| **精神层** | 思维/情感/集体无意识/模因 | 生命层涌现/独立场域/物理层干扰/高维投影 | — | 对下层因果力：无/单向影响/双向互动/完全控制 |
| **抽象层** | 概念/数学/命运/法则/叙事 | 人类建构/世界自带/高维存在/语言效应 | — | 具现化方式：不可见/符号承载/代理人显现/直接干涉物理 |

**层间互动规则**:
- **层间关系**: 严格分层（不可越界）/ 单向因果（上→下或下→上）/ 双向渗透 / 完全融合
- **跨层访问条件**: 修行/科技/死亡/梦境/顿悟/仪式

#### 1.1.2 个体性拓扑（Individuality Topology）

| 维度 | 可选值/描述 |
|------|------------|
| **自我边界定义** | 肉体皮肤 / 意识范围 / 社会角色 / 记忆集合 / 叙事连续性 |
| **他者承认机制** | 物理区分 / 灵魂标记 / 社会契约 / 语言回应 |
| **集体实体承认** | 国家/宗族/神/公司/AI集群 是否被承认为独立存在 |
| **分裂规则** | 人格分裂算几个个体 / 分身是否算同一存在 / 克隆体身份认定 |
| **融合规则** | 意识融合/记忆合并/社会身份合并后的个体性认定 |

#### 1.1.3 实在性状态（Reality Status）

| 参数 | 可选值 |
|------|--------|
| **世界本质** | 唯一实在 / 梦中梦 / 虚拟模拟 / 叙事层 / 多重现实叠加 |
| **多层结构** | 层数 / 层间通道 / 层间时间流速比 / 层间记忆保留规则 |
| **模拟假说** | 模拟者 / 模拟目的 / bug存在性 / 逃脱可能性 |
| **叙事自我意识** | 完全不自知 / 部分角色觉醒 / 作者介入 / 元叙事透明 |

### 1.2 因果架构（Causal Architecture）

描述事件与事件之间的因果律本身如何运作。

#### 1.2.1 因果方向性（Causal Directionality）

| 参数 | 可选值/描述 |
|------|------------|
| **时间箭头** | 单向 / 双向 / 局部可逆 / 循环 / 无时间 |
| **主导因果类型** | 物理因果 / 信息因果 / 意图因果 / 叙事因果 |

**因果类型清单**:
1. **机械因果**: 物理接触传递（推、拉、化学反应）
2. **信息因果**: 信息传递引发（知道→行动）
3. **意图因果**: 意图→行动（心→物）
4. **叙事因果**: 戏剧性需要（为了故事发展必须发生）
5. **共时因果**: 共时性（无因果关联的同时发生）
6. **逆因果**: 果影响因

#### 1.2.2 决定论谱系（Determinism Spectrum）

| 参数 | 可选值 |
|------|--------|
| **自由意志状态** | 完全自由 / compatibilism（兼容论）/ 软决定论 / 硬决定论 / 自由为幻觉 |
| **预言机制** | 自证（预言导致实现）/ 自否（预言导致避免）/ 模糊（多解）/ 绝对（无法改变）/ 观察者效应 |
| **命运结构** | 无命运 / 线性宿命 / 分支宿命（可选择的预定）/ 概率云 / 叙事宿命 |
| **干预点** | 关键节点可改变 / 任何时刻可改变 / 仅特定角色可改变 / 完全不可改变 |

#### 1.2.3 概率生态（Probability Ecology）

| 参数 | 可选值 |
|------|--------|
| **随机性来源** | 真随机 / 伪随机（底层决定）/ 叙事随机（作者需要）/ 混沌（敏感依赖） |
| **运气守恒** | 总量守恒（此消彼长）/ 个人运气池 / 无守恒 / 可转移 / 可窃取 |
| **巧合规则** | 禁止 / 允许小概率 / 关键转折依赖巧合 / 巧合实为隐藏因果 / 巧合被伪装成因果 |

#### 1.2.4 因果延迟（Causal Latency）

| 参数 | 可选值 |
|------|--------|
| **因果延迟** | 即时 / 短期延迟 / 长期延迟 / 跨代延迟 / 因果累积（量变→质变） |
| **因果记账** | 自动平衡 / 手动清算 / 延迟爆发 / 不记账（道德虚无）/ 叙事回收（戏剧性时刻清算） |

## 二、全局叙事约束框架（Meta-Narrative Global Context v2.1）

本层描述"叙事如何运作"和"世界如何运行"的元规则，分为 **Fabula（素材层）**、**Sujet（呈现层）**、**Text（文本层）**、**Contract（契约层）** 和 **Constraint（约束层级）**。

### 2.1 Fabula Mechanics（素材层）

故事素材在虚构世界中的因果-时间逻辑，与呈现方式（Sujet）解耦。

#### 2.1.1 事件本体（Event Ontology）

| 参数 | 可选值/描述 |
|------|------------|
| **事件粒度** | 原子事件（不可再分）/ 复合事件（可分解为子目标）/ 场景级事件（多角色交互） |
| **SRL模式** | 语义角色标注：谓词(动词) + 核心论元(Agent/Patient/Theme/Goal/Source) + 辅助论元(Instrument/Location/Time) |
| **物理事件** | 移动 / 战斗 / 制造 / 获取 / 破坏 / 感知 |

### 2.5 Constraint Hierarchy（约束层级）

区分紧约束、软约束和生成自由。

#### 2.5.1 硬约束（Hard Constraints）—— 违反即叙事崩坏

1. **契诃夫之枪**: 出现的细节必须回收（或明确标记为红鲱鱼）
2. **角色一致性**: 决策符合已建立的行为模式 / 能力包络
3. **世界规则完整性**: 已建立的物理/社会规则不可单方面修改
4. **因果完整性**: 已发生事件的因果链不可被事后否定

#### 2.5.2 软约束（Soft Constraints）—— 允许合理浮动

1. **模态平衡**: 叙述/对话/描写比例允许 ±10% 偏差
2. **风格温度**: 允许随情感峰值切换
3. **节奏弹性**: 允许在紧张场景加速、沉淀场景减速

#### 2.5.3 生成自由（Generative Freedom）—— 保留创造性空间

1. 词汇选择：同义词替换、句式变体
2. 比喻创新：在既定修辞密度内创造新比喻
3. 微观对话：具体措辞的创造性

## 四、校验与使用指南

### 4.1 提取方法

从已有章节中提取全局参数的方法：

1. **Fabula提取**：使用SRL（语义角色标注）识别谓词-论元结构，构建事件序列；对照Propp/Campbell功能表进行模式匹配。
2. **Sujet提取**：使用Genette的时间分析框架识别时序、时距、频率；分析视角锚定与切换规律。
3. **Text提取**：统计各章模态分布、句长分布、修辞密度、感官优先级，建立基线。
4. **Ontology提取**：通过角色对死亡、命运、知识、道德的态度反推底层世界观；通过事件因果链反推因果架构；通过时空描述反推时空本体论。
5. **Contract提取**：向测试读者呈现故事片段，调查其reader_position和emotional_contract感知。

### 4.2 校验清单

| 校验项 | 方法 | 通过标准 |
|--------|------|----------|
| **抽象纯度** | 扫描所有字段值 | 禁止出现任何专有名词（人名、地名、组织名、具体事件名、具体物品名） |
| **层间一致性** | 检查 `存在拓扑.层间互动` 与 `因果架构.因果方向` 是否矛盾 | 若层间为"严格分层"，则因果不可为"双向渗透" |
| **可执行性** | 每个底层规则转化为prompt约束 | 如"因果延迟:延迟爆发" → "当前章节埋下的因不得在当章回收" |
| **元叙事一致性** | 若 `第四墙:坚固`，则禁止角色感知读者/作者 | 全文扫描违规次数 = 0 |
| **叙事功能推进** | 检查每章是否推动功能序列 | 禁止跳过强制功能 |
| **守恒律检查** | 扫描死亡/资源/信息违规 | 死亡规则违规次数 = 0；信息凭空消失次数 = 0 |
| **模态稳定性** | 统计各章模态比例 | 标准差在基线 ±10% 内 |
| **预测有效性** | 用提取参数生成新章节，盲测风格一致性 | 读者评分 > 4/5 |
| **稳定性测试** | 增加新章节后重新提取全局信息 | 核心字段变化率 < 5% |

### 4.3 注入策略

**三层注入架构**：

```
System Prompt（常驻）
├── 底层世界观（Ontology）→ 作为元规则，全程不变
├── 全局叙事约束（Fabula/Sujet/Contract）→ 作为叙事法则，仅在发现设定矛盾时更新
└── 风格DNA（Text）→ 作为语言模板，仅在分析新章节后微调

User Prompt（每章动态）
├── 当前任务指令（功能位置、动力学状态）
├── 过程信息（最近3-5章摘要、角色状态、开放线索）
└── 本章特定约束（伏笔回收清单、悬念类型）
```

**双轨记忆架构**：
- **全局层（Global Layer）**：Agent初始化时加载，作为"作者人格"和"世界法则"固化在系统提示中，长期不变。
- **状态层（State Layer）**：每章续写前从数据库读取最新快照，续写完成后由Agent自动提取本章变化，回写更新状态表。
